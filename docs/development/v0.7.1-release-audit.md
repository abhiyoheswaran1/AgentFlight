# AgentFlight v0.7.1 Release Audit

- Audit date: 2026-06-21
- Package version before bump: `0.7.0`
- Intended release version: `0.7.1`

## Implementation Summary

`v0.7.1` is a documentation and release-asset patch. It refreshes the README
terminal hero so the visible workflow matches the current handoff-first product
flow:

- `agentflight init`
- `agentflight start`
- `agentflight verify`
- `agentflight status`
- `agentflight handoff`
- `agentflight history --limit 1`

The patch also updates README samples and release notes so `handoff` is the
primary local review packet and `history` is the way to reopen recent local
artifacts.

## Scope Assessment

The release is acceptable as a patch because:

- The change is limited to README copy, demo asset source, generated terminal
  GIF, release notes, task evidence, and package metadata.
- No runtime TypeScript behavior changed.
- No dependencies changed.
- No public CLI contracts changed.
- No secret-handling, auth, billing, database, cloud, telemetry, or hosted
  behavior changed.
- Local verification, package dry-run, audit, ProjScan doctor/preflight, and
  AgentLoopKit verification passed before commit.

## Dogfood Note

The temporary package smoke initially ran in a non-git directory and correctly
warned that `status` and `handoff` could not inspect changed files. The
git-backed smoke then passed. The smoke fixture also showed that a repo without
`node_modules/` ignored will produce noisy changed-file review focus; this is
not a release blocker for the README hero patch.
