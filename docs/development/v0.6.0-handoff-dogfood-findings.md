Dogfood date: 2026-06-20

Local build tested: `node dist/cli.js --version` returned `0.6.0`.

Code point tested: `b3d9b87` (`Add local AgentFlight handoff workflow`).

This pass tested the unpublished local `agentflight handoff` workflow before any
next version cut. No package version bump, tag, publish, or release was done.

## Summary

The local handoff workflow is useful enough for a release audit. In both tested
repos, a clean verification path produced a ready handoff with a clear next
action and artifact list. A controlled failing verification produced a blocked
handoff, exited non-zero, preferred stderr for the displayed failure excerpt, and
kept raw stdout and stderr evidence intact.

The main product friction is still first-run workspace noise in real apps:
`.agentflight/config.json` is correctly visible as project config, while
`.projscan-memory/memory.json` appears as an unknown changed file with an
appropriate suggestion to add `.projscan-memory/**` to
`changedFileFilters.ignore` when the file is not meant for review. That behavior
matches the current design, but it remains the clearest next ergonomic issue.

## Bugs Found

No concrete release-blocking bugs were found in the local handoff workflow.

Small issues and product friction:

1. First-run `.projscan-memory/memory.json` still ranks above project config in a
   fresh real app because it is an unknown changed file. The guidance is correct,
   but first-run workspace hygiene remains the next high-value product polish.
2. In repo-local dogfood, AgentFlight handoff review focus naturally surfaced
   AgentLoop task files created by the dogfood itself. This is expected, but it
   reinforces that the handoff is only as useful as the current changed-file set.
