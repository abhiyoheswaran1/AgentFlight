# AgentFlight v0.12.0 Release Audit

Audit date: 2026-06-24

Package version before bump: `0.11.0`

Intended release version: `0.12.0`

## Implementation Summary

AgentFlight v0.12.0 makes the Project Review Contract more repo-calibrated and
more explicit about stale proof:

- Repo-calibrated proof guidance compares the current session's proof with
  similar local ready handoffs and suggests additional proof commands when the
  current proof is weaker than local history.
- Proof freshness attribution shows which files and changed-file categories made
  proof stale.
- Docs-only stale changes stay in manual-review guidance instead of forcing
  verification reruns when no proof-required files changed.
- Status, handoff, Markdown report, HTML replay, resume, and status JSON expose
  the same calibration and freshness signals.
- A dogfood regression was fixed so a historical failed verification no longer
  marks Project Review Contract proof as failed after a later matching pass.

The release stays local-first and source-free. It does not add hosted review, PR
comments, CI policy enforcement, named verification profiles, source upload,
telemetry, login, billing, dashboards, or GitHub App behavior.

## Scope Assessment

This is a minor release because it improves the main review decision path across
existing local review surfaces.

No dependency changes were made.
