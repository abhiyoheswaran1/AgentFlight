Audit date: 2026-06-21

Intended release version: `0.8.0`

## Implementation Summary

AgentFlight v0.8.0 adds proof freshness as a local-first review contract layer:

- `agentflight verify` stores source-free changed-file proof snapshots on verification runs.
- Review Intelligence compares the latest passing proof snapshot with the current changed-file state.
- Review surfaces now distinguish `current`, `stale`, `covered`, `missing`, `failed`, `not required`, and `unknown` proof.
- Stale proof creates an actionable proof gap and keeps readiness at `Needs verification`.
- Status, Markdown reports, HTML replays, resume prompts, snapshots, handoffs, and history-derived readiness agree on proof freshness.

The implementation stores repo-relative paths, file state, file size, and SHA-256 hashes. It does not store source text, diffs, telemetry, cloud state, or uploaded evidence.
