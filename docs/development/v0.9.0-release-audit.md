# AgentFlight v0.9.0 Release Audit

Audit date: 2026-06-22

Package version before bump: `0.8.0`

Intended release version: `0.9.0`

## Implementation Summary

AgentFlight v0.9.0 adds a deterministic local Review Contract claim ledger on
top of the v0.8.0 proof-freshness layer:

- Review Intelligence now emits explicit claims for the session task, changed
  files, proof gaps, and readiness state.
- Claims can be `supported`, `needs_review`, `unsupported`, `failed`, `stale`,
  `not_testable`, or `unknown`.
- Status JSON, terminal status, Markdown reports, HTML replays, resume prompts,
  and handoffs now surface Review Contract claims.
- The contract remains local-only and source-free: it uses repo-relative paths,
  proof-gap IDs, commands, status labels, and short reasons, not source text,
  diffs, telemetry, cloud state, or uploaded evidence.

The release candidate also includes two dogfood fixes found during the Review
Contract pass:

- `agentflight doctor` no longer warns about missing root `test`, `build`,
  `typecheck`, or `lint` scripts when configured `.agentflight/config.json`
  verification commands exist. This avoids misleading setup warnings in
  monorepos that verify through subpackage commands.
- Review Intelligence now treats passed `verify` scripts such as
  `npm run verify` as proof, so AgentFlight's own configured verification
  command can satisfy source and test proof gaps.

## Scope Assessment

This is a minor release because it adds a new review artifact surface and
structured Review Intelligence output. It does not add cloud, login, billing,
hosted behavior, PR comments, JSON/CI expansion, named verification profiles,
or export modes.

No dependency changes were made.
