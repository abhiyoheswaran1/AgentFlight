# AgentFlight v0.11.0 Release Audit

Audit date: 2026-06-24

Package version before bump: `0.10.0`

Intended release version: `0.11.0`

## Implementation Summary

AgentFlight v0.11.0 makes the Project Review Contract explainable across the
local review workflow:

- Project Review Contract requirements now show why each rule matched, which
  changed-file categories activated it, which proof command satisfied it, and
  which review actions remain.
- Status, Markdown reports, HTML replays, resume prompts, and handoffs now show
  the same required-proof decision path.
- Handoff now leads with a decision and why summary before the detailed review
  packet.
- Review Contract claims now include Project Review Contract requirement claims
  before file-level claims.
- README and packaged docs now explain the local contract path and include a
  Project Review Contract decision visual.

The release keeps AgentFlight local-first. It does not add hosted review, PR
comments, CI policy enforcement, named verification profiles, source upload,
telemetry, login, billing, dashboards, or GitHub App behavior.

## Scope Assessment

This is a minor release because it changes the main review workflow and rendered
review artifacts. The scope stays on one product spine: local proof expectations,
actual proof, and remaining review work.

No dependency changes were made.
