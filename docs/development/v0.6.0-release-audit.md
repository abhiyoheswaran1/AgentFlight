# AgentFlight v0.6.0 Release Audit

Audit date: 2026-06-19

Package version before bump: `0.5.1`

Intended release version: `0.6.0`

## Release Rationale

This is a minor release because the post-v0.5.1 work adds user-facing local
review surfaces and workflow ergonomics:

- replay navigation and review anchors
- first-run workspace hygiene guidance
- optional ProjScan review hints for deterministic ranking
- named verification profiles
- compact Markdown report mode
- local PR-comment draft report mode
- local JSON status output
- AgentLoopKit evidence filtering in AgentFlight changed-file surfaces

The release remains local-first. It does not add cloud, hosted review, login,
billing, telemetry, source upload, GitHub App behavior, or automatic PR posting.

## Implementation Summary

AgentFlight v0.6.0 improves the local review loop after v0.5.1:

- Reviewers can navigate long replay ledgers faster.
- Developers can run repeated local proof command groups through config-defined
  verification profiles.
- Scripts can read status as structured local JSON.
- Reports can be generated as full, compact, or local PR-comment-draft
  Markdown artifacts.
- Review Intelligence can accept typed ProjScan hints from callers without
  coupling to ProjScan execution.
- Generated AgentLoopKit evidence no longer pollutes AgentFlight's own
  changed-file review surfaces, while task contracts and policy files remain
  visible.

## ProjScan Sign-Off

ProjScan is expected to return a manual release-signoff caution for this release
because the accumulated handoff is large and includes several command, renderer,
core, docs, and test changes.

The caution is acceptable only if the final release checks show:

- ProjScan doctor health remains `100/A`.
- ProjScan preflight/review report no concrete blockers.
- No dependency changes.
- No contract changes.
- No new taint or dataflow risks.
- No new cycles.
- No risky functions.
- Full verification and package dry-run pass.

Manual release sign-off: accepted as a scale/review caution, not a concrete
release blocker, if the final command outputs match the criteria above.

## Staging Notes

Stage intended product, docs, tests, release audit, and version files only.
Local AgentLoop run ledgers and handoff artifacts are release evidence, not npm
package content, and should not be staged unless explicitly needed by repo
policy.

## Deferred

- Login, billing, Pro/Team, or account features.
- GitHub App behavior.
- Automatic PR comment posting.
- Broader CI integration beyond local JSON status.
