# AgentFlight v0.4.0 Dogfood Findings

Dogfood date: 2026-06-14

Published package tested: `agentflight@latest`, confirmed as `0.4.0`.

## Summary

AgentFlight v0.4.0 Review Intelligence is useful and understandable in real local workflows. The strongest result is that docs-only changes with proof evidence are treated calmly: low risk where appropriate, no proof gaps, and a clear ready-for-review state.

The main friction is first-run/generated-file noise. In repos without an existing AgentFlight config, `.agentflight/config.json` appears as a changed file, remains visible as intended, and often ranks ahead of the actual docs change because it is classified as `unknown`. ProjScan also creates `.projscan-memory/memory.json`; the new configurable filter works, but users need to know to add `.projscan-memory/**`.

The biggest correctness bug remains interrupted verification. A terminated long-running verification left an orphan `verification_started` event without a failed verification run. Status still reported `1 passed, 0 failed`, which can hide that an attempted proof command never finished.

## Review Intelligence Findings

- Review focus ranking is easy to understand when the changed files are normal project files.
- Docs-only changes are not over-scored when AgentFlight config/generated files are not present.
- Proof gaps are useful when they point to an exact configured command.
- Readiness is appropriately conservative for real proof states, but it currently ignores orphan `verification_started` events.
- Status, report, replay, and resume generally agree with each other on review focus and readiness.
- Existing v0.4.0 changed-file filters work, but the user has to configure generated tool artifacts.
- Output is understandable without knowing internals, except for first-run `.agentflight/config.json` and `.projscan-memory/memory.json` noise.

## False Positives

- `.agentflight/config.json` ranked above simple README edits in both ProjScan and fifa-predictor. It is intentionally visible, but classifying it as `unknown` makes it look more suspicious than it is.
- `.projscan-memory/memory.json` ranked above the actual app README change until `.projscan-memory/**` was configured.
- ProjScan report output showed a missing `npm test` verification line after an interrupted command, but Review Intelligence still said no proof gaps.

## False Negatives

- Interrupted verification was not reflected in readiness. The session had an orphan `verification_started` event for `npm test`, but status reported `1 passed, 0 failed` and Ready for review.
- Long-running verification gave no heartbeat or progress output before it was terminated.

## Readiness Model Feedback

- About right for AgentFlight docs-only work after proof.
- About right for fifa-predictor after typecheck/test proof.
- Too loose when a verification command was started but never completed. Orphan started events should block or warn readiness.
- Ready-for-review wording is clear.

## Proof Gap Feedback

- Proof gaps were understandable where they appeared in v0.4.0 release smoke tests.
- For docs-only changes, no proof gaps is reasonable when proof exists.
- Report still includes older verification-gap text separate from Review Intelligence proof gaps. This creates two proof-gap concepts and should be unified or clearly labeled.

## Changed-File Filter Feedback

- Runtime `.agentflight/sessions`, `.agentflight/reports`, `.agentflight/current`, and `.agentflight/evidence` stayed filtered.
- `.agentflight/config.json` stayed visible as intended.
- `.projscan-memory/**` should remain user-configurable, but AgentFlight should suggest it when ProjScan memory appears.
- Configuring `.projscan-memory/**` works.
- Generated files must be configured before generating final reports/replays, otherwise existing artifacts preserve the noisy file list.

## Replay/Report/Resume Feedback

- Replay is much more useful after v0.4.0 because review focus, proof gaps, readiness, and timeline are visible together.
- Report is useful, but the old "Missing passing verification evidence for" line can conflict with Review Intelligence's "Proof gaps: none."
- Resume is useful enough to hand to another agent. It includes changed files, review focus, proof gaps, readiness, next action, and guardrails.
- Status is the clearest artifact for quick triage.

## Bugs Found

1. Interrupted verification can leave an orphan `verification_started` event without a failed run.
2. Readiness does not account for orphan verification events.
3. Report can show legacy verification gap text that disagrees with Review Intelligence proof gaps.

## UX Friction

- Long-running verification has no heartbeat/progress output.
- First-run `.agentflight/config.json` can dominate review focus.
- `.projscan-memory/memory.json` appears as `unknown` unless configured.
- `doctor` warnings are useful, but can distract in repos that intentionally lack a script such as `lint` or `typecheck`.
- `agentflight start` can create AgentLoopKit task side effects when AgentLoopKit is present.

## Recommended v0.4.1 Patch Items

1. Detect orphan `verification_started` events and surface them in status/report/replay/resume as incomplete verification.
2. Make incomplete verification block or warn readiness.
3. Add heartbeat/progress output for long-running `agentflight verify` commands.
4. Unify legacy verification gaps with Review Intelligence proof gaps in reports.
5. Classify `.agentflight/config.json` as AgentFlight project config instead of generic `unknown`.
6. Suggest adding `.projscan-memory/**` to `changedFileFilters.ignore` when that path appears.

Patch candidate status:

- Addressed in the unreleased v0.4.1 patch candidate.
- `.projscan-memory/**` remains user-configured; AgentFlight suggests it when `.projscan-memory/memory.json` appears.
- `.agentflight/config.json` remains visible, but is classified as AgentFlight project config.
