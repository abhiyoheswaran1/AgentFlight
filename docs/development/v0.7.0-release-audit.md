## Release Rationale

This is a minor release because the branch adds user-facing local review workflow
capabilities on top of the already published v0.6.0 release:

- `agentflight handoff` for a local review packet that leads with readiness,
  proof gaps, review focus, failed verification excerpts, and artifact paths.
- `agentflight history` for reopening recent local session artifacts without
  adding search indexing, sync, export, or session switching.
- Handoff-first ready review guidance across status, resume, history, report,
  replay, and docs.
- Replay Review Path guidance for faster review of long local evidence ledgers.
- First-run proof seeding and workspace-hygiene copy that keep generated local
  tool state visible but below real review targets.

The release remains local-first. It does not add cloud, hosted review, login,
billing, telemetry, source upload, GitHub App behavior, automatic PR comments,
database behavior, or hosted features.

## Implementation Summary

AgentFlight v0.7.0 turns the v0.6.0 review surfaces into a more complete local
handoff workflow:

- Reviewers can open one handoff packet first, then drill into report, replay,
  and resume artifacts as supporting evidence.
- Engineers can list recent local sessions and reopen the most useful artifact
  without scanning `.agentflight/` directories by hand.
- Status and resume now point at existing local handoff/report/replay artifacts
  instead of asking users to regenerate artifacts that already exist.
- Review Intelligence separates unresolved failed verification from historical
  failed runs that later passed.
- Concurrent verification and artifact generation paths preserve local evidence
  more reliably.
- Public copy now consistently uses coding-agent and agentic-engineering
  positioning rather than assistant-style language.

- No contract changes.

- No risky functions.

Manual release sign-off: accepted as a scale/review caution, not a concrete
release blocker, if the final command outputs match the criteria above.
