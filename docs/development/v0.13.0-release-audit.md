# AgentFlight v0.13.0 Release Audit

Audit date: 2026-06-24

Package version before bump: `0.12.0`

Intended release version: `0.13.0`

## Implementation Summary

AgentFlight v0.13.0 improves the existing local review workflow instead of
adding a new surface:

- Trust Delta summarizes what changed in the review trust state.
- Review Queue orders proof reruns, missing proof, manual checks, calibration
  guidance, and file inspection.
- Local Review Receipts let a handoff be accepted and later marked stale when
  the work changes.
- Role-aware review routing gives Maintainer, Verification, Docs/DX, Security,
  and Release reviewers a scoped local review path.
- Project Review Contract proof selection is more accurate for stale proof,
  current proof, accepted proof alternatives, and accepted-handoff boundaries.
- Status, handoff, report, replay, resume, history, and JSON output now agree
  more consistently on readiness, proof state, stale receipts, and review
  actions.
- Dense review surfaces now cap large Review Focus and changed-file lists while
  keeping full details available in the deeper artifacts.
- Markdown, replay, and terminal display paths strip unsafe display controls or
  escape active markup without modifying raw stdout/stderr evidence files.
- Verification evidence paths now validate persisted session ids before local
  artifact writes.
- Passing verification commands with large output no longer false-fail at
  Node's default 1 MiB buffer.
- Clean-worktree resume restoration uses a direct local artifact copy instead
  of a read/write transform.

The release remains local-first and source-free. It does not add hosted review,
PR comments, CI policy enforcement, JSON/CI expansion, named verification
profiles, source upload, telemetry, login, billing, dashboards, or GitHub App
behavior.

## Scope Assessment

This is a minor release because it materially improves the main review decision
path across existing local review surfaces.

No dependency changes were made.
